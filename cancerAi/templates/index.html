Project Overview

Breast Cancer

Breast cancer starts when cells in the breast begin to grow out of control. These cells usually form a tumor that can often be seen on an x-ray or felt as a lump. The tumor is malignant (cancer) if the cells can grow into (invade) surrounding tissues or spread (metastasize) to distant areas of the body. Breast cancer occurs almost entirely in women, but men can get breast cancer, too. Cells in nearly any part of the body can become cancer and can spread to other areas.

Why CNN and not Regular Neural Networks(FC)

Fully connected neural networks are good classifiers, however they aren't good for feature extraction. Before the emergence on CNNs the state-of-the-art was to extract explicit features from images and then classify these features. CNNs are trained to identify and extract the best features from the images for the problem at hand. That is their main strength. The latter layers of a CNN are fully connected because of their strength as a classifier. CNN do not try to understand the entire problem at once. It looks at the input piece by piece, derives low level features, through many layers, hierarchically. These helps the deep learning models to learn complicated patterns, by gradually building them from simpler ones. This helps them to not only ‘see’ the features but also see the hierarchy of how these features are built upon.

Hidden Layers/Feature Extraction

• Convolution is one of the main building blocks of a CNN. In the case of a CNN, the convolution is performed on the input data with the use of a filter or kernel (these terms are used interchangeably) to then produce a feature map.
• We execute a convolution by sliding the filter over the input. At every location, a matrix multiplication is performed and sums the result onto the feature map. After performing numerous convolutions on our input, all these different feature maps are created and are put together as the final output of the convolution layer.
• ReLU layer (activation) is used to bring non-linearity to the network. It sets all the negative pixels to zero and performs element wise operation. The original image is scanned in multiple Convolution and ReLU layers for locating hidden features and patterns in the image. The output is a Rectified Feature Map.

Hidden Layers/Feature Extraction

Because the size of the feature map is always smaller than the input, we have to do something to prevent our feature map from shrinking. This is where we use padding. This ensures that each pixel in the image is given an opportunity to be at the center of the filter. This is often not a problem for large images and small filters but can be a problem with small images. It can also become a problem once a number of convolutional layers are stacked. The ‘padding‘ value of ‘same‘ calculates and adds the padding required to the input image (or feature map) to ensure that the output has the same shape as the input. Here stride is the step the convolution filter moves each time. A slide 1 means filter slides pixel by pixel.

Hidden Layers/Feature Extraction

This Rectified Feature Map is fed as input to the pooling layer, which basically reduces the dimensionality of the feature map. The output is a Pooled Feature Map.Function of pooling is to continuously reduce the dimensionality i,e reduce the number of parameters ansd computation in the network. Maxpooling takes the maximum value in each window, our size is 2*2. This decrease the feature map while at the same time keeping the significant information. In this model, Spatial pooling is carried out by five max-pooling layers, performed after some of the convolutional layers.

Classification

We Fatten the layers to convert 3D into 1D vector to pass through softmax. The softmax is normally applied to the last layer in a neural net. It converts the output of the last layer of your neural network into what is essentially a probability distribution.

Epochs

An epoch is an iteration over the entire data set, as defined by steps per epoch, which in turn is the total number of steps to yield from generator before declaring one epoch finished and starting the next epoch.
